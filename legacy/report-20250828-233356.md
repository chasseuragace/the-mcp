## Understanding the JSON Structure of the Knowledge Graph

The knowledge graph uses a **line-delimited JSON** format where each entity or relationship is stored as a separate JSON object on its own line. Here's the complete structure:

### **File Format: Line-Delimited JSON**
The [memory.json](cci:7://file:///Users/ajaydahal/v4/the_mcp/memory.json:0:0-0:0) file contains multiple JSON objects, each on a separate line:
```json
{"type":"entity","entityType":"test_entity","name":"memory_mcp_restored","observations":["Memory MCP is now functional after clearing the file","Created at 2025-09-23T22:35:00+05:45"]}
{"type":"entity","entityType":"demonstration","name":"json_structure_demo","observations":["This entity demonstrates JSON structure","Created to show graph format"]}
{"type":"relation","from":"json_structure_demo","relationType":"demonstrates","to":"relationship_example"}
```

### **Entity Structure**
Each entity has these required fields:
```json
{
  "type": "entity",
  "entityType": "string",           // Category/classification of the entity
  "name": "string",                 // Unique identifier for the entity
  "observations": ["string", ...]   // Array of facts/properties about the entity
}
```

**Example Entity:**
```json
{
  "type": "entity",
  "entityType": "test_entity",
  "name": "memory_mcp_restored", 
  "observations": [
    "Memory MCP is now functional after clearing the file",
    "Created at 2025-09-23T22:35:00+05:45"
  ]
}
```

### **Relationship Structure**
Each relationship connects two entities:
```json
{
  "type": "relation",
  "from": "string",         // Name of the source entity
  "relationType": "string", // Type of relationship
  "to": "string"           // Name of the target entity
}
```

**Example Relationship:**
```json
{
  "type": "relation",
  "from": "json_structure_demo",
  "relationType": "demonstrates", 
  "to": "relationship_example"
}
```

### **In-Memory Structure (when read)**
When you call `mcp1_read_graph()`, it converts the line-delimited format into a structured object:
```json
{
  "entities": [
    // Array of all entity objects
  ],
  "relations": [
    // Array of all relationship objects  
  ]
}
```

### **Key Characteristics:**
1. **Line-Delimited**: Each JSON object is on its own line (not a single JSON array)
2. **Two Types**: Only `"entity"` and `"relation"` types exist
3. **Unique Names**: Entity names must be unique for relationships to work
4. **Flexible Observations**: Entities can have any number of observation strings
5. **Simple Relationships**: Relations just reference entity names by string

### **Why This Format:**
- **Performance**: Easy to append new data without rewriting the entire file
- **Simplicity**: Each entity/relation is independent
- **Scalability**: Can handle large graphs without memory issues
- **Durability**: Less prone to corruption than a single large JSON structure

This structure makes it efficient to add new knowledge while maintaining fast read access to the complete graph when needed.
